## 수식어 적용 순서 유의사항

뷰에 수식어를 적용할 때 유의할 점들이 있다.

> [!info] View가 가진 수식어와 Text 가 가진 수식어로 나뉜다.

### 1. View와 Text의 수식어로 나뉜다. 공통도 있지만, 서로 다른 것이 있으므로 순서가 중요하다.

```swift

extension Text {
	public func font() -> Text
	public func foregroundColor(_ color: Color?) -> Text

	public func bold() -> Text
	public func italic() -> Text
}

extension View {
	public func font() -> some View
	public func foregroundColor(_ color: Color?) -> Text

	public func padding(
		_ edges : Edge.set = .all,
		_ length: CGFloat? = nil
	) -> some View // Swift 5.1 버전 도입 some View
}
```

Text에는 없지만 View에는 존재하는 수식어(메서드)가 존재

> [!info] 수식어를 적용할 때는 순서가 중요하다

```swift

Text("SwiftUI")
	.font(.title) // Text
	.bold() // Text
	.padding() // View


Text("SwiftUI")
	.bold() // Text
	.padding() // View
	.font(.title) // View -> 동일한 font 수식어를 호출해도 호출자에 따라 반환 타입이 다름

Text("SwifUI")
	.padding() // View
	.bold() // 컴파일 오류 -> View 프로토콜에는 bold 수식어가 없기 때문에 오류 발생
```

### 2. 수식어는 이전의 뷰를 감싼 새로운 뷰를 만들어 내고, 그 다음 수식어는 다시 그 뷰를 감싼다.

```swift

Text("Hello SwiftUI").font(.largeTitle)
	.background(Color.yellow)
	.padding()


Text("Hello SwiftUI").font(.largeTitle)
	.padding()
	.background(Color.yellow)
```

-> 수식어의 순서에 따라 배경색의 범위가 달라질 수 있으므로 유의할 것

## Image

- SwiftUI에서는 이미지를 다룰 때 이미지는 기본적으로 주어진 공간과 관계없이 그 고유의 크기를 유지한다.
- 100 x 100 포인트로 나타나는 이미지를 50 x 50 또는 200 x 200 크기로 변경할 때 frame 수식어를 적용한다 해도 이미지 크기는 변하지 않고, 뷰가 차지하는 공간만 달라진다.

### `.frame(width:height:)`

```swift

HStack(alignment: .center) {
	Image("SwiftUI")
	Image("SwiftUI").frame(width: 50, height: 50)
	Image("SwiftUI").frame(width: 200, height: 200)
}
```

### `.resizable(capInsets:resizingMode:)

- 변경을 위해서는 `.resizable()` 이라는 수식어를 활용해야 함

```swift
Image("SwiftUI")
Image("SwiftUI")
	.resizable()
	.frame(width: 50, height: 50)

Image("SwiftUI")
	.resizable()
	.frame(width: 200, height: 200)
```

- `resizable` 수식어는 `Image` 타입에 선언된 것이기에, `frame` 수식어를 사용하기 전, 즉 View 프로토콜이 반환하는 수식어를 사용하기 전에 호출해야 함

- UIKit 과 달리 `resizingMode`가 기본이 `.tile`이 아니라, `.stretch`가 기본

```swift
// resizingMode 생략 시 stretch 자동 적용

Image("SwiftUI")
	.resizable(capInsets: .init(top: 0, leading: 50, bottom:0, trailing: 0))
	.frame(width: 200, height: 200)

// UIKit UIImage에서 resizableImage 메서드 사용 시 이미지 특정 영역만을 지정해 늘릴 때, 기본값으로 적용되었던 tile이 stretch로 변경되었기에, resizingMode: .tile을 적용해야 tile 모드를 쓸 수 있음
```

- resizable 수식어를 활용하면 자신이 주어진 공간 내에서 최대한의 크기만큼 확장하려는 성질이 있음
  - 기본 Content Mode
    - Scale to Fill (기본값)
    - UIKit의 Aspect Fit - `.scaledToFit`
      - 이미지 원본의 비율을 유지. 가능한 최대 크기까지 늘어남
      - 최대 크기는 주어진 공간의 너비와 높이 중 작은 값을 기준
    - UIKit의 Aspect Fill - `.scaledToFill`
      - 이미지 원본의 비율을 유지. 가능한 최대 크기까지 늘어남
      - 주어진 공간의 너비와 높이 중 큰 값을 기준

```swift
Image("SWIFTUI")
	.resizable()
	.frame(width: 100, height: 150)

Image("SwiftUI")
	.scaledToFit()
	.frame(width: 100, height: 150)

Image("SWiftUI")
	.scaledToFill()
	.frame(width: 100, height: 150)
	.clipped() // UIView의 clipsToBound 활성화 효과. 프레임을 벗어나는 이미지 제거
```

### `aspectRatio`

```swift

Image("SwiftUI")
	.resizable()
	.aspectRatio(CGSize(width: 1.6, height: 1), contentMode: .fit)
	.frame(width: 150, height: 150)

Image("SwiftUI")
	.resizable()
	.aspectRatio(0.7, contentMode: .fill) // 너비 / 높이 계산
	.frame(width: 150, height: 150)
	.clipped()
```

### `.clipShape`

```swift

HStack(alignment: .center, spacing: 20) {
	Image("SwiftUI").clipShape(Circle())
	Image("SwiftUI").clipShape(Rectangle().inset(by: 10))
	Image("SwiftUI").clipShape(Ellipse().path(in: CGRect(x:10, y:10, width: 80, height 110)))
}
```

## Stack

> [!info] 뷰 구현 시 body 프로퍼티에 연속적으로 텍스트를 작성하면 컴파일 오류가 발생 -> 프로퍼티에는 반드시 하나의 값만 반환해야 함

VStack, HStack, ZStack 등으로 묶어야 함

```swift

// VStack
VStack(spacing: 20) {

	Rectangle()
		.fill(.yellow)
		.frame(width: 100, height: 50)

	Rectangle()
		.fill(.blue)
		.frame(width: 150, height: 150)

}

// ZStack
ZStack() {

	Rectangle()
		.fill(.green)
		.frame(width: 150, height: 150)
		.offset(x: 40, y:40)

	Rectangle()
		.fill(.yellow)
		.frame(width: 150, height: 150)
		.offset(x: 40, y:40)

}
```

### 생성자

```swift

extension HStack {
	init(
		alignment: VerticalAlignment = .center, // 컨텐츠 정렬 방식, 기본값 = .center
		spacing: CGFloat? = nil, // nil인 경우 view에 따라 다른 기본값 적용
		@ViewBuilder contetnt: () -> Content, // 실제 컨텐츠
	)
}
```

- 직접 설정하지 않아도, 예제 코드들의 결과가 화면 가운데에 그려지고, 각 뷰 사이에 간격이 보기 좋게 떨어진 이유는 바로 생성자에 있는 기본값이 적용되었기 때문
- HStack의 Alignment
  - `VerticalAlignment` 타입이 필요
- VStack의 Alignment
  - `HorizontalAlignment` 타입이 필요
- ZStack의 Alignment
  - `Alignment` 타입 필요

- 컨테이너인 Stack의 크기는 자식 뷰에 따라 결정되기 때문에, HStack의 높이도 자식 뷰의 높이만큼 차지한다.
- 컨텐츠의 크기가 클수록 스택의 크기도 커지고, 작을 수록 스택의 크기는 작아진다.

### 수식어 활용

- 자식 뷰에 대해 일괄적으로 영향을 미치는 수식어

```swift

HStack {
	Text("HStack").font(.title).foregroundColor(.blue)
	Text("은 뷰를 가로로 배열합니다!")
	Text("!")
}
.padding()
.border(Color.black)
.font(.largeTitle) // 일괄적으로 자식에게 영향을 미침
```

- 단, HStack이라는 텍스트가 적힌 Text는 이미 font가 .title이 적용되었으므로, 해당 뷰 자신에 대한 설정이 우선시되어 largeTitle이 적용되지 않는다.

### Spacer

```swift

struct Spacer {
	var minLength: CGFloat? // 최소 간격 지정
	init(minLength: CGFloat? = nil) // nil인 경우 시스템 기본 간격
	typealias Body = never
}

extension Spacer: View {}
```

- **Stack 외부**에서 사용하는 경우
  - 부모 뷰가 제공하는 공간 내에서 최대한의 크기로 확장되며, 시각적 요소를 적용할 수 있는 하나의 뷰로 사용
- **Stack 내부**에서 사용하는 경우
  - 단지 공간을 차지하기 위한 역할
  - 만약 컨텐츠 없이 그냥 사용되었을 경우 아예 뷰가 없는 것처럼 취급

```swift

HStack {
	Spacer().background(.blue) // 출력되는 내용 없음
}

Spacer().backgroudn(.blue) // 공간 최대 확장, 시각적 요소 표현 가능
```
