## 1. Preview

**아래 상황 제외하고, 어떠한 작은 변화라도 일어나는 순간 자동 갱신이 중단됨**

- 파일의 Top Level
- 구조체 / 클래스의 구현 범위에서
  - 키워드
  - 속성
  - 프리프로세서 구문 수정

**하지만, 함수나 메서드 범위 안에서 코드를 변경하는 경우**

- 모듈을 다시 빌드하지 않고, 대부분의 상황에서 자동 갱신 기능의 혜택을 누릴 수 있음
- 연산 프로퍼티(Computed Property) 는 입력값이 없는 함수와 동일하기 때문에 함수와 메서드처럼 마찬가지이다.
  - body 연산 프로퍼티 내에서 코드를 수정하는 경우, 프리뷰를 다시 갱신 하지 않고도 수정사항이 잘 반영됨\

### 프리뷰 수식어 살펴보기

```swift
// xcode 13-14 버전
struct ContentView_Previews: PreviewProvider {
	static var previews: some View {
		ContentView()
	}
}


// xcode 15+, iOS17+ 아래 코드 권장
#preview {
	Preview()
}
```

## 2. Environment Values

- 뷰를 구성하는 데 필요한 각종 환경 설정과 관련된 정보를 EnvironmentValues 타입이 관리한다.

```swift
struct EnvironmentValues: CustomStringConvertible { ... }
```

- colorScheme, timeZone, locale, calendar, layoutDirection, sizeCategory, undoManager 등 기존에 UITraitCollection을 비롯해 다양한 클래스에서 나뉘어 사용하고 관리되던 속성들을 이제 EnvironmentValues 하나에 모두 담아서 쉽게 접근하고 관리할 수 있음
- 프레임워크에 의해 별도 관리되므로, 어떤 뷰에서든 쉽게 접근할 수 있다. - 이 때 얻게 되는 값은 상위 계층의 뷰가 가진 환경 요소를 그대로 상속받게 된다. - 윈도우 또는 윈도우 루트 뷰와 같은 최상위 뷰에서 적용된 값은 모든 자식 뷰에 영향을 주게 된다. - 단, 하위 계층에 있는 뷰에서 개별적으로 다른 환경을 구성했다면 그 뷰에 속한 자식 뷰들을 변경된 값을 우선하여 사용
  e.g. ColorScheme
- 최상위 뷰에 .dark로 적용한 경우 그 자식 뷰들을 자동으로 다크 모드가 된다.
- 하지만 자식 뷰 중에 하나에 .light 값을 주면, 해당 뷰와 그 뷰의 서브 트리는 라이트 모드가 기본값이 된다.

### environment 수식어

```swift
Text("Hello, SwiftUI").frame(width: 100)
	.environment(\.font, .title) // 1
	.environment(\.lineLimit, 1) // 2
	.environment(\.minimumScaleFactor, 0.5) // 3
```

- 일반 수식어와 차이가 없어 보이지만, environment 수식어는 자식 뷰에까지 영향을 미치는지 해당 뷰에만 영향을 미치는지에 달려 있다. 해당 뷰의 서브 트리 전체에 영향을 미친다.

### @Environment

```swift
//
//  HelloSwiftUIApp.swift
//  HelloSwiftUI
//
//  Created by KWON HYUKMIN on 5/11/26.
//

import SwiftUI

@main
struct HelloSwiftUIApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .environment(\.locale, .init(identifier: "ko_KR"))
                .environment(\.colorScheme, .light)

            ContentView()
                .environment(\.locale, .init(identifier: "en_US"))
                .environment(\.layoutDirection, .rightToLeft)
                .environment(\.colorScheme, .dark)
        }
    }
}

```

> [!info] \.locale의 \는 keyPath로 보통 Environment.locale하면 프로퍼티에 접근해 실제 값을 얻어내지만, \는 locale 프로퍼티를 가리키는 **참조 경로**를 의미한다.
>
> 이전에 다룬 Identifiable 프로토콜을 채택한 PersonRow 구조체도 List View에 표시할 때 id를 활용하여 ForEach 구문에서 Text를 배열할 때 사용하였다.

### Custom Environment

```swift
// EnvironmentKey -> Protocol

struct MyEnvironmentKey: EnvironmentKey {
	static let defaultValue: Int = 0
}

extension EnvironmentValues {
	var myEnvironment: Int {
		get { self[MyEnvironmentKey.self] }
		set { self[MyEnvironmentKey.self] = newVlaue }
	}
}
```

## 2. Identifiable

### Equatable

값 타입에서 Equatable 프로토콜을 통해 단지 상태 같음(동등성)을 통해 대상을 구분

### Identifiable

동일성까지 판단 -> List, ForEach 로 리스트 구현 시, id 를 요구함 (id가 같은 경우 같은 아이템으로 바라보는 문제)

- 저장 프로퍼티로 값 저장

```swift
struct Animal: Identifiable {
	let id: UUID = UUID()
}
```

- 연산 프로퍼티로 다른 프로퍼티의 값 지정

```swift
struct Animal: Identifiable {
	var id: UUID { uuid }
	var uuid: UUID = UUID()

```

- 기존의 id가 이미 존재할 경우, 프로토콜 채택

```swift
struct Animal {
	var id: UUID
}
extension Animal: Identifiable ()
```

> 위 두 프로토콜을 모두 준수하는 경우, Identifiable을 개체의 식별자로 사용하기
> id를 통해 개체를 구분하고, 각 개체의 상태 변화를 측정가능
> 값 타입에도 참조타입과 같은 정체성을 가질 수 있도록 하기
